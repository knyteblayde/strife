Custom Domain names

custom domain names are the way of naming your application like it is actually a live website,
for example, in your local development server you are used to be developing on a URL like http://localhost/myproject/index.php but when we use a custom domain name, we can do it like so http://website.dev or http://website.com or any URL that you would want to have for your application.
custom domains aren't just to fancify our URL but it also solves the directory issues that is encountered in procedural programming, you'll see the purpose of these configurations later on. but for now, let's proceed.

We will start to configure our httpd-vhosts.conf file,
for Xampp, navigate to C:\xampp\apache\conf\extra\httpd-vhosts.conf and for Wampserver, C:\wampp\apache\conf\extra\httpd-vhosts.conf (please note that my installation directory might be different to yours) open the file using any text editor, and add this line of code at the end of the file:

<VirtualHost *:80>
    DocumentRoot "C:/xampp/htdocs/spiral/public"
    ServerName myapp.dev
</VirtualHost>
Notice the "C:/xampp/htdocs/spiral/public", this should be the directory where you extracted/installed the Spiral Framework.
while myapp.devwill be our custom domain name for this purpose, but feel free to change that to whatever you like.

Now save and exit, let us now configure hosts file
Open your Notepad with administrator privilleges(run as administrator) and open C:\Windows\system32\drivers\etc\hosts and add this line at the end: 
127.0.0.1  myapp.dev
If you followed all the above steps, you will now be able to visit http://myapp.dev in your web browser.
