Routing

Basic Routing logic

Routing is one of the most common thing on MVC frameworks, you assign a route for a specific page, a route for when saving data, or when you want to delete an item and etc.

Spiral's basic routing logic is very simple, for example suppose we want to view all products and our URL might look like this: 

http://myapp.dev/products/index
controllers rely on what is with the URL, to nail it down, what happends in the background is Spiral looks for the first string on the URL after the domain name, in this case, it is products 
http://myapp.dev/ products /index
Spiral then will handle this string, it will look for a class ProductsController ('Products' is our string value from the URL and 'Controller' is automatically append) inside the controllers directory.
Now for the second string on the URL is the index 
http://myapp.dev/products/ index
The string index will then be the method inside the ProductsController

Spiral's default controller

inside app/controllers directory, there's a php file called HomeController.php this is the default controller class for handling URLs like http://myapp.dev and http://myapp.dev/home/index it looks like this: 

<?php

class HomeController
{
    public function index()
    {
        return View::render('frontend/index');
    }
}
Go and edit the index() method like this: 
<?php

class HomeController
{
    public function index()
    {
        echo "Hello, world!";
    }
}
by visiting http://myapp.dev or http://myapp.dev/home/index it will echo what we have typed inside index method.
